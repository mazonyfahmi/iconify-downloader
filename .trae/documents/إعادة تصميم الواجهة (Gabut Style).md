# إعادة تصميم واجهة المستخدم لتطابق Gabut Download Manager

بناءً على تحليلي للمشروع (Electron + React + Tailwind) والصورة المرفقة، سأقوم بإعادة تصميم الواجهة بالكامل لتتحول من نمط "تطبيق هاتف" إلى "تطبيق سطح مكتب كلاسيكي".

## التغييرات الهيكلية (Architecture Changes)
1. **استبدال التخطيط الحالي (`Layout.tsx`)**:
   - إزالة الشريط السفلي (`Bottom Navigation`).
   - إنشاء هيكل جديد: `Toolbar` (أعلى) -> `InfoBar` (معلومات) -> `MainContent` (وسط) -> `StatusBar` (أسفل).

## المكونات الجديدة (New Components)
### 1. شريط الأدوات العلوي (Top Toolbar)
- **الإجراءات اليسرى**:
  - أيقونة **إضافة (+)**: لفتح نافذة البحث عن الأيقونات (Search Modal).
  - أيقونة **تشغيل (Play)**: لبدء تحميل الأيقونات المختارة.
  - أيقونة **إيقاف (Pause)**: (عنصر بصري حالياً).
  - أيقونة **تنظيف (Broom)**: لمسح القائمة المختارة.
- **العنوان**: في المنتصف "Gabut Download Manager" (أو اسم التطبيق).
- **الإجراءات اليمنى**: إعدادات، بحث سريع.

### 2. شريط المعلومات (Info Bar)
- شريط رفيع أسفل الأدوات يعرض:
  - عدد الأيقونات المختارة.
  - حالة التحميل (السرعة/الحجم - سنستخدم بيانات محاكية أو حقيقية إن وجدت).

### 3. منطقة المحتوى الرئيسية (Main Content)
- **الحالة الفارغة**: تصميم مطابق للصورة "No File Download" مع إمكانية السحب والإفلات.
- **قائمة التحميل**: عند اختيار أيقونات، تظهر هنا كقائمة (List View) بدلاً من الشبكة (Grid) لتشبه مدير التحميلات.

### 4. شريط الحالة السفلي (Bottom Status Bar)
- فلاتر التبويب: (All, Downloading, Paused, Complete, Error) مع مربعات اختيار (Checkboxes) وأعداد.

## خطة التنفيذ (Implementation Steps)
1. **إنشاء ملفات المكونات**: `Toolbar.tsx`, `InfoBar.tsx`, `StatusBar.tsx`, `MainList.tsx`.
2. **تعديل `App.tsx`**:
   - دمج المكونات الجديدة.
   - تحويل واجهة "البحث" الحالية إلى نافذة منبثقة (Modal) تظهر عند الضغط على "إضافة".
   - ربط زر "بدء" بوظيفة التحميل الموجودة في `DownloadView`.
3. **التنسيق (Styling)**:
   - استخدام Tailwind CSS لضبط الألوان (الأخضر للأزرار، الخلفية البيضاء، النصوص الرمادية).
   - ضمان استجابة الواجهة وتطابقها مع الصورة.
